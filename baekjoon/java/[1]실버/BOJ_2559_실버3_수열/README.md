## 첫 번째 풀이 방법

- `슬라이딩 윈도우` 로 일정 크기의 구간 만큼 이동해주며 합을 구했다.
- 합을 구한 다음 max와 비교를 통해 계속해서 갱신해주었다.
- 하지만 시간이 1008ms로 굉장히 길게 나왔다. (시간 초과는 x)
- 이중 for문 때문인 것 같아 for문을 하나만 쓰는 방식을 고민했다.


<br>

## 두 번째 풀이 방법
- `누적합` 을 통해 문제 풀이
- 입력을 받을 때 누적합을 통해 배열을 갱신하고 구간합을 구할 때 for문 안에서 `max = Math.max(max, arr[i] - arr[i-K]);` 위 방식을 통해 최대 구간합을 갱신해주었다.
- for문을 하나만 쓰니까 208ms로 1/5 만큼 줄일 수 있었다!! 👍
